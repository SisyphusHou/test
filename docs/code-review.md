## 代码质量问题

### P0 — 应立即修复
- **var/let 混用**: 全文件 var 和 let 交替使用，作用域隐患

### P1 — 建议本迭代完成
- getBestMove() 中 topMoves 随机选择范围受限（topThreshold=3 忽略 0.95 阈值的走法）
- minimax() 函数过长 (~60行，圈复杂度 ~15)
- 候选走法评分逻辑在 getBestMove() 和 minimax() 中重复

### P2 — 可维护性
- SCORE_TABLE 键值语义不明确，防御加权系数 1.05 无解释
- isDraw() 用 moveHistory.length >= 225 而非检查空位数
